群組指令捷徑

使用捷徑

雖然有上一章的指令別名，但群組指令的使用仍稍嫌不方便，因此我們引進了「群組指令捷徑」，可以讓你像使用一般指令一樣使用群組指令。
我們先來看一下 music 指令群清單： 正常來講必須使用 z!m join 才能使用 music 的 join 指令，但群組指令捷徑允許你直接使用 z!join 來執行。

原理

讓我們看看在接收到 z!join 後，HiZollo 到底做了什麼：首先他看到你想要執行 join 指令，便去他的指令庫尋找 join 指令，但是他找不到，這時他會去找所有的指令群，詢問這些指令群他們有沒有群組指令叫做 join？而 music 有，於是 HiZollo 就把指令轉交給 music 處理，也就等同於執行 z!music join。
同理，你也可以使用 z!best 去執行 osu 的 best 指令。

別名適用原則

使用快速用法時，是可以使用群組指令的別名的。舉例來說 osu 的 best 指令，有個別名叫做 bp，所以你也可以直接使用 z!bp 來執行這個指令。

主指令優先原則

當群組指令的快速用法剛好跟某個主指令的用法相同時，將會優先執行主指令，而非去執行該群組指令。例如 diep 的 server 指令和一般的 server 指令是同名的，所以當你執行 z!server 時，他會執行一般的 server 指令，而非 diep 的 server 指令。 但是，diep 的 server 指令有個別名叫做 discord，所以如果你使用 z!discord 的話，因為沒有叫做 discord 的主指令， 所以他就會去執行 diep 的 server 指令。

撞名時的處理

如果今天有兩個指令群，它們剛好有同名的群組指令，這時當你使用該群組指令捷徑時，HiZollo 會跳出一個清單問你要執行哪個指令群的群組指令。選擇後 HiZollo 便會執行選擇的那個群組指令。

z!help 查不到

當你使用像是 z!help play，他會告訴你那個指令不存在。這是因為 HiZollo 並沒有 play 指令，只有 music 這個子指令管轄者有。z!play 中的這個 play 並不是它的正式名稱，而是一個簡短的用法，並不能取代原本的名稱，這點跟別名是不一樣的。
